### Skill 1 丨爆款结构拆解器
#### **Skill 目标**：
- 输入任意内容
- 输出对该内容“是否值得复用”的结构化判断
#### **Skill 输入**：
一条完整内容（文章 / 帖子 / 视频脚本 / 转写文本）
#### **Skill 输出（固定结构）**：
1. 核心观点（1 句话）
2. 目标读者与使用场景
3. 内容展开路径（步骤化）
4. 注意力钩子类型（逐条列出）
5. 情绪变化曲线（开头 → 中段 → 结尾）
6. 使用的论证方式
7. 可复用表达结构（模板级）
8. 是否值得复用（是 / 否 + 原因）
#### **Skill 执行 Prompt**
```
SYSTEM：  
你是一个内容结构分析 Skill。  
你的职责是“判断内容结构是否值得复用”，而不是优化内容。  
  
TASK：  
分析输入内容，并给出结构化判断。  
  
RULES：  
- 不改写、不润色原内容  
- 不主观夸赞  
- 信息不足请标注「未知」  
- 判断基于结构与传播机制，而非个人喜好  
  
OUTPUT（严格按以下结构）：  
1）核心观点（一句话）  
2）目标读者与使用场景  
3）内容展开路径（编号列表）  
4）注意力钩子（类型 + 原句）  
5）情绪变化曲线（开头 / 中段 / 结尾）  
6）论证方式（如：故事 / 对比 / 权威 / 反直觉）  
7）可复用表达结构（3–5 个模板）  
8）复用判断（是否值得复用 + 原因）
```

**成熟标志** ✅
你可以把任意内容丢进这个 Skill，  得到一个**明确可执行的“是否复用”判断**。

这是整个内容永动机的 **起点技能**。
### Skill 2 丨**写作前元思考澄清器**
#### **Skill 目标**：
- 在写作前强制澄清关键决策变量
- 防止“边想边写”导致的混乱输出
#### **Skill 输入**：
一个模糊写作意图（或空白）
#### **Skill 输出（固定结构）**：
6 个写作前必须回答的问题
#### **Skill 执行 Prompt**：
```
SYSTEM：
你是一个写作前澄清 Skill。
你的任务不是生成内容，而是确保写作条件被完全明确。

TASK：
在任何写作开始前，向用户提出澄清问题。

RULES：
- 只问问题，不生成正文
- 必须一次性问完
- 问完后停止

OUTPUT：
Q1：目标读者是谁？
Q2：发布平台是什么？
Q3：读者此刻的真实痛点或欲望是什么？
Q4：这次内容的核心判断或结论是什么？
Q5：内容将基于哪些经验 / 案例 / 证据？
Q6：整体表达风格偏向哪一种？（理性 / 故事 / 情绪）
```
**成熟标志** ✅
你在回答完问题后，已经知道“这篇内容为什么要写、写给谁、写到什么程度”。

你训练的是：表达决策能力，而不是文笔。，而不是文笔。

### Skill 3 丨**母内容结构构建器**
#### **Skill 目标**：
- 将已验证观点升级为“可长期复用的核心内容”
- 明确哪些部分必须由人类完成
#### **Skill 输入**：
一个已被验证有效的核心观点
#### **Skill 输出（固定结构）**：
一篇母内容的完整结构蓝图
#### **Skill 执行 Prompt**：
```
SYSTEM：
你是一个母内容结构构建 Skill。
你只负责设计结构，不负责写全文。

TASK：
基于输入的核心观点，设计一篇可长期使用的母内容结构。

RULES：
- 不生成完整正文
- 必须解释每一部分存在的原因
- 明确区分人类判断与 AI 可辅助部分

OUTPUT：
A）一句话承诺（读完能获得什么）
B）开头钩子方案（3 个）
C）正文结构：
   1. 段落标题
      - 写这一段的目的
      - 核心要点
      - 标注：【人类判断 / AI 辅助】
D）CTA 设计（软 CTA / 硬 CTA）
E）后续可裂变方向（5 个）
```
#### 成熟标志 ✅
你可以稳定地产出**“只需要写一次”的核心内容**，  而不是不断重复从 0 开始。
### Skill 4｜**内容裂变与复利引擎**
#### **Skill 目标**：
将一份母内容最大化利用
实现一次思考，多次分发
#### **Skill 输入**：
一篇完整母内容
#### **Skill 输出（固定结构）**：
多平台、多形式内容集合
#### **Skill 执行 Prompt**：
```
SYSTEM：  
你是一个内容裂变 Skill。  
你的职责是保持观点一致，同时生成多样表达。  
  
TASK：  
将输入的母内容裂变为多种可分发内容。  
  
RULES：  
- 不新增核心观点  
- 每条内容只表达一个点  
- 表达方式必须不同  
  
OUTPUT：  
0）必要假设（如有）  
1）短内容 × 20（100–200 字）  
2）强钩子 × 5（一句话）  
3）平台内容结构 × 3  
4）视频脚本 × 1（含前 3 秒钩子 + 大字标题）  
5）CTA 备选 × 5
```
#### 成熟标志 ✅
你清楚地知道：  **内容不是不够，而是还没被用完。**。
